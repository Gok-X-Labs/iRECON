# Security Policy

## Reporting a Vulnerability

If you discover a security issue in iRECON, please report it responsibly.

- Do NOT disclose the issue publicly
- Contact: Gokul (gosint26@gmail.com or github.com/Gok-X-Labs)

---

## Scope

Relevant issues include:

- Violations of Safe Processing Mode
- Unexpected network interactions
- Data leakage or unsafe handling
- Incorrect parsing leading to unsafe behavior

---

## Safe Processing Guarantee

iRECON is designed to:

- Never contact attacker infrastructure
- Never execute payloads
- Process all data in memory
- Use intelligence APIs only (server-side)

Any deviation from this is considered a critical issue.

---

## Response Process

- Issues will be reviewed as soon as possible
- Fixes will be prioritized based on severity
- Responsible disclosure is appreciated

---

## Disclaimer

iRECON is a passive analysis tool and is not intended for active exploitation, scanning, or offensive use.
